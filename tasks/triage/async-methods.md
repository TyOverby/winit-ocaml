# Async Methods Need Manual Implementation

## Problem
Several WebGPU methods use async callbacks which cannot be auto-generated by the current code generator. These need either:
1. Manual synchronous wrappers (current approach)
2. Generator support for async callbacks

## Affected Methods
- `instance.request_adapter` - requests adapter asynchronously
- `adapter.request_device` - requests device asynchronously
- `device.pop_error_scope` - pops error scope with callback
- `queue.on_submitted_work_done` - notifies when work completes
- `buffer.map_async` - maps buffer asynchronously
- `shader_module.get_compilation_info` - gets compilation info via callback

## Current Workaround
These methods have synchronous wrappers implemented manually in the low-level C stubs or high-level OCaml code.

## Possible Solutions
1. Keep manual sync wrappers (current)
2. Add async callback support to generator
3. Expose both sync and async versions
