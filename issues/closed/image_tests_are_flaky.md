# Image tests are flaky

Every time that I run `dune exec test/test_compute.exe`, the `render_clear.png`
and `render_triangle.png` files that it generates show up as being different to
git.  I can't see a difference between them, but it's annoying to always be
updating them.

Could you look into why they differ, and fix the source of nondeterminism?

## Analysis

The PNG files generated by `test_compute.ml` contain embedded timestamps that
change every time the test runs. Examining the PNG file with `xxd` shows:

1. `tIME` chunk - contains the current time when the image was created
2. `tEXt date:create` - creation timestamp
3. `tEXt date:modify` - modification timestamp
4. `tEXt date:timestamp` - timestamp

These are added by ImageMagick's `convert` command in the `ppm_to_png` function
(line 23-29 of `test/test_compute.ml`).

## Plan

1. Modify the `ppm_to_png` function to use the `-define png:exclude-chunks=date,time`
   flag which tells ImageMagick to exclude timestamp-related chunks from the PNG output.

## Validation Criteria

1. Running `dune exec test/test_compute.exe` twice in a row should produce
   identical PNG files (no git diff)
2. The build should still succeed (`dune build`)
3. The tests should still pass
4. The PNG files should still be valid and viewable
